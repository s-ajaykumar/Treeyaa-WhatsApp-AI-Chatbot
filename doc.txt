Latest:

Used sarvam api to detect all languages and convert the audio into the user language text.
Set language code in the api to "unknown" to detect all languages.

Prompt change:
* If quantity type is not given or don't match with the Unit, ai will clarify with the user about the Unit of the product. Like it will ask we sell tomatoes in Kg but you asked 3pcs tomatoes.
* Think field is used.
* Two fields - "TranslatedName" and "Language" are added to the success response template.
User may speak in a language and db contains items in English.
"TranslatedName" will contain item names in the user asked language.
"Language" will contain the user language.
"Name" will contain the item name in db as usual.
* While showing options for an item, item names in user language will also be shown within brackets below each item option.
* Added two types - "clear_cart", "place_order". Whenever the user asks to clear the cart (or) place the order, it will be returned.
* PLACE_ORDER is removed from handling rules.
* Removed words - "fruits" and "veg" from treeyaa details.
* While showing OutOfStock and NoMatch item names in success response, translated names will also be shown along with it.
